# VEX Part Manager

A browser-based CAD analyzer for VEX Robotics designs. Drop in a CAD file and
the app will:

- Render an interactive 3D model you can rotate, zoom, and click.
- Detect each part using a heuristic geometry classifier (c-channels, angles,
  plates, shafts, gears, sprockets, wheels, standoffs, spacers, screws, nuts,
  V5 motors / brain / battery, pneumatics, ...).
- Group parts by category in a sidebar with quantities and direct order links
  to [VEX Robotics](https://www.vexrobotics.com) and
  [Robosource](https://www.robosource.net).
- Play an animated **explode view** that fans each part out into its category
  group, with a soft outline of the original assembly remaining visible so you
  can see where each part came from.
- Export the bill of materials as CSV.

## Supported file types

| Format        | Notes                                                              |
| ------------- | ------------------------------------------------------------------ |
| `.stl`        | Binary or ASCII. Connected components are split into separate parts.|
| `.obj`        | Each `o` / `g` block is treated as a part.                          |
| `.step` / `.stp` | Tessellated via `occt-import-js` (OpenCascade WASM, loaded lazily). |

The classifier expects the file to be in inches, millimeters, centimeters, or
meters. The **Units** dropdown defaults to **Auto-detect** and works for most
VEX assemblies; switch it manually if your file uses an unusual scale.

## How parts are identified

For each separable mesh in the assembly the classifier computes:

- Sorted bounding-box dimensions in inches
- Mesh volume and surface area
- "Tubeness" (cylinder-like) and "discness" (gear/wheel-like) shape scores
- Bounding-box fill fraction

Each part in `src/partsdb.js` declares an inch-based shape profile and an
optional shape hint (`isDisc`, `isCylinder`). The mesh is matched against
every profile and assigned the best fit, provided the confidence exceeds a
threshold; otherwise it is grouped under **Unidentified** with its dimensions
shown so you can identify it manually.

Because the database uses search URLs rather than fixed SKUs, the resulting
order links remain valid even when VEX / Robosource rotate their catalogs.

## Caveats

- This is a heuristic classifier, not a CAD library — always verify the
  generated parts list before ordering.
- STEP files require the OpenCascade WASM build (≈5 MB) to be reachable from
  the CDN. If you are offline, export your CAD as STL instead.
- The app runs entirely client-side; no CAD data leaves your machine.

## Tests

Two headless-Chrome smoke tests live under `scripts/`:

```bash
npm install        # installs ws for the CDP harness
npm run test       # runs both the smoke and end-to-end tests
```

- `scripts/headless-check.mjs` — boots the page, asserts the sidebar
  populates its category list, and surfaces any console errors.
- `scripts/e2e-check.mjs` — uploads `fixtures/sample.stl`, verifies parts are
  detected and classified, triggers the explode animation, and asserts the
  slider reaches 100%.

The fixture STL is generated by `scripts/make-fixture.mjs`. It contains a
synthetic c-channel, shaft, four standoffs, a gear disc, and a spacer.

## Run locally

The site is a pure static page using ES modules and an import map — no build
step required. Start any static file server in the repo root:

```bash
npm run serve      # python3 -m http.server 8000
# or
npx serve .
```

Then open <http://localhost:8000>.
